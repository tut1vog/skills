Takes a discussion — a file, pasted text, or a scoped thread from the current session — and emits a reusable **context skill** written to `.claude/skills/<slug>/SKILL.md` so future agents can load the captured knowledge without re-reading the original discussion.

## Gate

Skip and tell the caller why when either applies:
- A skill covering the same discussion already exists in a loaded skill or `CLAUDE.md`.
- The input is too thin to extract signal (fewer than ~10 substantive exchanges or ~200 words of discussion content).

## Input modes

Detect the mode from the invocation:
- **File path** — user provides a path (e.g. `discussions/auth-design.md`). Use `Read` to load the file.
- **Inline text** — user pastes the discussion alongside the invocation. Use the pasted content directly.
- **Current conversation** — no file or pasted text; user provides a scoping hint (e.g. "the discussion about authentication above"). Use the hint to focus on the relevant portion of the active context window. If no scoping hint is given, ask for one before proceeding.

## Tools

`Read` (file path mode), `Bash` (for `pwd` when writing output).

## Phase 1: Ingest

Load the discussion content per the detected input mode. If the input is a file path and the file cannot be read, stop and report the error rather than guessing at content.

## Phase 2: Analyze

Extract four components from the discussion:

1. **Context / Background** — the problem or situation that motivated the discussion.
2. **Decisions Made** — what was resolved and the reasoning behind each decision.
3. **Key Insights** — facts or observations that surfaced and were not obvious before the discussion.
4. **Open Questions** — threads that remain unresolved. Each item gets an inline staleness note (see template).

Do not invent decisions or insights not present in the source. If the discussion is ambiguous, capture the ambiguity in Open Questions rather than forcing a resolution.

## Phase 3: Propose slug

Derive a slug from the discussion content: lowercase, hyphens only, 3–5 words, must end with `-context` (e.g. `auth-middleware-rewrite-context`).

Show the proposed slug and a one-sentence rationale. Wait for user confirmation or an alternative before writing anything to disk.

## Phase 4: Write the context skill

1. **Run `pwd`** via Bash to get the absolute project root. Use that result — **never** `~/.claude/`.
2. **Write `<pwd-result>/.claude/skills/<slug>/SKILL.md`** using the template below.
3. **Split to `REFERENCE.md`** if the full content would exceed 200 lines — move verbose background or raw excerpts there and add a one-line pointer in `SKILL.md`.

### Generated SKILL.md template

```
---
name: <slug>
description: <one sentence on what this skill covers>. Context only — load to gain knowledge captured from a discussion on <topic>.
---

> **Context skill.** This skill provides context only. Read and internalize it; do not prompt the user for a next action. Acknowledge with exactly: "Context loaded."

## Context / Background

<the problem or situation that motivated the discussion>

## Decisions Made

- **<decision>** — <reasoning>

## Key Insights

- <insight>

## Open Questions

> **Note:** These questions were unresolved as of the discussion date. Verify current status before acting on them.

- <open question>

---
*Generated by discussion-to-context-skill on <YYYY-MM-DD>. Decisions and insights reflect the state of the discussion at that point in time.*
```

## Constraints

- Do not implement, fix, or build anything — analyze and write the skill file only.
- Do not commit the generated files.
